How I Accidentally Discovered a Critical Vulnerability in Easy Anti-Cheat

idkhidden [2024-10-28]

Introduction

In this write-up, I’ll walk you through the steps that led to my discovery of a critical vulnerability in Easy Anti-Cheat.

Discovery

It all started when I was exploring the EAC launcher to understand its security mechanisms. I wondered how difficult it would be to protect my own application. This curiosity led me to start reversing their launcher. After spending a solid day on this, I concluded that there were no proper certificate checks in place. With this and a proper configuration of settings.json, we can easily exploit the vulnerability.

Exploit

First, I removed the original game application and renamed my application to match the game's executable. In this case, since I was exploiting Fortnite, I renamed my application to FortniteClient-Win64-Shipping.exe. Since the EAC launcher module doesn't check the validity of the certificate in the Easy Anti-Cheat folder. Using this knowledge, I completely removed the certificates from the folder. Next, I turned to the settings.json file, where I found the correct settings that not only validated but also bypassed the checks, thus successfully completing the exploit. The settings.json file was as follows: 

        {
            "title"                                          :"Fortnite",
            "executable"                                    : "FortniteClient-Win64-Shipping.exe",
            "productid"                                     : "prod-fn",
            "sandboxid"                                     : "fn",
            "deploymentid"                                  : "62a9473a2dca46b29ccf17577fcf42d7",
            "requested_splash"                              : "EasyAntiCheat/SplashScreen.png",
            "wait_for_game_process_exit"                    : "false",
            "hide_bootstrapper"                             : "false",
            "hide_gui"                                      : "false"
        }

Now, if we launch the Easy Anti-Cheat launcher, our application will function seamlessly under EAC protection.

Impact

Bypassing EAC protection allows an attacker to run their own application under the guise of a legitimate game process. This could lead to several security impacts, including:

Attackers could gain access to sensitive resources or functionalities within the protected application, which are typically restricted.

The vulnerability could be exploited to execute malicious software disguised as a game client, potentially compromising users’ systems.

Attackers could potentially gather sensitive information from users or the environment that the application operates in.

Conclusion

After finding the vulnerability, I immediately reported it to Epic Games. Unfortunately, upon submission, I was informed that my report was considered a duplicate of a similar issue raised in 2023, yet it has not been fixed. Anyways Thank you for reading my write-up!
